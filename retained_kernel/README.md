# Retained Kernel Model

This package adds a general retained-feedback kernel to
`EvolvedCooperation`. It is a non-website-backed experimental module that
extracts the abstract routing logic behind `retained_benefit/` into more
general trait-and-identity language.

The intended role is simple:

> `retained_benefit/` is the website-facing named model
>
> `retained_kernel/` is the more general kernel written as an explicit module

## Purpose

The existing `retained_benefit/` package already studies a broad claim about
cooperation, but its naming still frames the mechanism as one specific model.
This package makes the underlying kernel explicit.

It asks:

> when a trait creates local output, and some fraction of that output is
> retained for identity-matching recipients, under what conditions does the
> trait spread?

That is the same local-selection logic as the retained-benefit module, but
with more neutral terminology so it can be reused or compared without assuming
the website-facing presentation.

## Core Rule

Each lattice site carries:

- one continuous trait value `t in [0, 1]`
- one inherited identity label

At each step, site `i` produces:

<p><code>O<sub>i</sub> = a &times; t<sub>i</sub></code></p>

That output is split into:

<p><code>O<sub>i</sub><sup>retained</sup> = r &times; O<sub>i</sub></code></p>
<p><code>O<sub>i</sub><sup>open</sup> = (1 - r) &times; O<sub>i</sub></code></p>

The producer also pays:

<p><code>K<sub>i</sub> = c &times; t<sub>i</sub></code></p>

Fitness is then:

<p><code>F<sub>i</sub> = w<sub>0</sub> + received_open<sub>i</sub> + received_retained<sub>i</sub> - K<sub>i</sub></code></p>

Variable definitions:

- <code>t<sub>i</sub></code>: trait value at site <code>i</code>
- <code>O<sub>i</sub></code>: gross output generated by site <code>i</code>
- `a`: `trait_output_scale`
- `r`: `retention_fraction`
- <code>K<sub>i</sub></code>: private cost paid by site <code>i</code>
- `c`: `trait_cost_scale`
- <code>F<sub>i</sub></code>: resulting fitness used in local replacement
- <code>w<sub>0</sub></code>: fixed `base_fitness`
- <code>received_open<sub>i</sub></code>: open output received from the full local neighborhood
- <code>received_retained<sub>i</sub></code>: retained output received only from producers whose identity matches site <code>i</code>

The update step is synchronous:

1. Every site distributes open and retained output.
2. Fitness is computed from received output minus private cost.
3. Each site copies one local parent with probability proportional to fitness.
4. The copied trait mutates with probability `mutation_rate`.
5. Identity is inherited from the copied parent.

## Relation To `retained_benefit/`

Mechanically, the two packages are deliberately aligned:

- `retained_benefit.cooperation` corresponds to `retained_kernel.trait`
- `retained_benefit.lineage` corresponds to `retained_kernel.identity`
- `retained_benefit_fraction` corresponds to `retention_fraction`
- the neighborhood routing and local replacement rules are the same

The difference is framing:

- `retained_benefit/` is the public named interpretation used in the repo's
  website-facing module set
- `retained_kernel/` is the explicit general kernel for direct experimentation
  inside the Python repo

## Package Contents

- `retained_kernel_model.py`
  Main runtime, retained-kernel update rule, logging, and module entry point.
- `retained_kernel_pygame_ui.py`
  Live Pygame grid viewer with trait and identity views.
- `config/retained_kernel_config.py`
  Active runtime configuration and primary source of truth for the main run.
- `utils/matplot_plotting.py`
  Small Matplotlib summary figure for completed runs.

## Quick Start

Run from the repository root:

```bash
./.conda/bin/python -m retained_kernel.retained_kernel_model
```

For the live grid viewer:

```bash
./.conda/bin/python -m retained_kernel.retained_kernel_pygame_ui
```

Normal workflow:

1. Edit `config/retained_kernel_config.py`.
2. Run the module from the repo root.
3. Inspect the JSON output at `retained_kernel/data/latest_run.json`.

Viewer workflow:

1. Run `./.conda/bin/python -m retained_kernel.retained_kernel_pygame_ui`.
2. Use `Space` to play or pause the model.
3. Use `S` or the right-arrow key for one synchronous step.
4. Use `V` to toggle between trait intensity and inherited identity structure.
5. Use `T` for the trait view and `I` for the identity view.
6. Use `+` and `-` to change playback speed.

## Current Scope

The package currently keeps the same narrow mechanism family as
`retained_benefit/`:

- local lattice interaction
- one continuous inherited trait
- one inherited identity marker
- one protected-return parameter `retention_fraction`
- one synchronous local replacement rule
- one live grid visualization comparable to the retained-benefit viewer

It does not yet add alternative retention kernels, nonlocal routing, explicit
movement, or multiple competing retention rules. Those would be natural next
extensions if this module becomes the base for further experiments.
